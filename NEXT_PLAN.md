# 下一步计划

## 局内浮窗战术显示

目标：把战术、路线强弱、资源节奏、装备建议等 Coach 信息扩展到局内浮窗中显示。

风险检查：实现前必须确认 Riot / LoL 对第三方工具、Overlay、实时战术提示、自动化输入、读取游戏内存/网络数据等规则边界，避免被判定为作弊或违规工具。

初步原则：
- 只显示玩家可见信息和赛前/本地分析结果。
- 不读取游戏内存。
- 不拦截或修改客户端/游戏进程。
- 不自动操作鼠标键盘。
- 不提供躲技能、自动走位、自动连招、实时敌方隐藏信息等功能。
- 优先做“手动/赛前策略提示”，谨慎做实时局内决策提示。

## 合规搜索结论（2026-07-10）

结论：局内浮窗本身不一定会被视为作弊；Riot 明确表示使用 API、game client、in-game APIs 的 overlays/internal tools 理论上可继续工作。但必须避免读取内存、自动化操作、敌方隐藏信息、敌方大招计时、自动决策等会造成不公平优势的功能。

可做方向：
- 显示赛前/BP阶段已经生成的战术摘要。
- 显示玩家自己选择打开的装备路线、阵容提醒、资源节奏清单。
- 使用官方/公开/本地缓存数据。
- 如果接入 Live Client Data API，只读取 Riot 文档公开的本地 HTTPS 数据，并限制为玩家可见信息。

高风险/禁止方向：
- 读取游戏内存或注入游戏进程。
- 自动追踪敌方不可见技能、召唤师技能、大招冷却。
- 根据隐藏信息给出实时行动指令。
- 自动操作鼠标键盘、自动走位、自动连招、脚本化操作。
- 修改客户端、修改游戏数据、绕过反作弊。

实现建议：先做“局内静态战术卡片 / 手动查看”，不要做实时敌方状态追踪。

官方依据：
- Riot 第三方应用支持页面说明：第三方应用会被 Vanguard 持续扫描，并需要符合第三方政策。
- Riot 开发者政策允许基于 Riot API、游戏客户端、局内 API 的 overlay / internal tool / app 继续工作，但必须遵守产品政策、API 规则和公平性边界。
- Riot 政策明确禁止通过产品帮助规避系统、泄露隐藏信息、破坏公平竞技、操纵或自动化用户输入等行为。

下一步安全实现方案：
1. 局内浮窗只读取 `data/live_state.json` 中 BP 阶段已经生成/冻结的战术结论。（第一版已完成）
2. 局内显示内容优先限定为：阵容优劣、路线强弱、资源节奏、团战目标、装备路线。
3. 不做敌方实时冷却追踪、不读内存、不注入、不自动点击。
4. 先提供“手动打开 / 可关闭 / 可拖动 / 透明度调整”的静态战术浮窗。
5. 后续如接入 Live Client Data API，单独做合规审查，只使用 Riot 公开接口和玩家可见信息。
